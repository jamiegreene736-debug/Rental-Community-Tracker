
### Key Business Rules (DO NOT CHANGE)

**Pricing Formula:**
```
net_required   = buy_in_nightly × 1.20          # 20% markup target
airbnb_listed  = net_required / 0.97            # Cover Airbnb's 3% host fee
vrbo_listed    = net_required / 0.92            # Cover VRBO's 8% host fee
booking_listed = net_required / 0.85            # Cover Booking.com's 15% commission
```

**6BR = Two 3BR Logic:**
- The 6BR Poipu listing requires buying TWO 3BR condos in Poipu Kai simultaneously
- We scrape availability for 3BR condos from Airbnb + VRBO
- For each 14-day window, if fewer than 3 available 3BR condos exist → BLACKOUT
- The threshold is 3 (not 2) to create a buffer against fake/stale availability

**Availability Buffer Logic:**
```
available_count = raw_search_results × 0.70     # 30% discount for fake availability
if min(available_count_unit_0, available_count_unit_1) < 2:  → HARD BLACKOUT
if min < 3 (configurable MIN_AVAILABLE_PER_SOURCE):           → SOFT BLACKOUT
if min >= 3:                                                  → OPEN
```

## WHAT THIS SYSTEM DOES (summary)

Every **6 hours**: Scrapes nightly rates from:
- Airbnb (unofficial v3 API + v2 fallback)
- VRBO (REST API + GraphQL fallback)
- Google Search (SERP API → extracts prices from snippets and pages)
- Property Management Companies in Poipu/Kauai (direct site scraping)

Every **4 hours**: For all 14-day windows across the next 24 months:
- Checks how many 3BR condos in Poipu Kai are available on Airbnb + VRBO
- Applies 70% fake-availability discount
- Determines OPEN or BLACKOUT per window

Every **1 hour**: Pushes to Guesty:
- Calendar blocks for unavailable windows
- Nightly prices (Airbnb-optimized with correct host fee markup)

Every **24 hours**: Full deep scan of all of the above

All results are stored in a SQLite database (`vacation_rental.db`) for audit, trend analysis, and failure recovery.

## IMPORTANT NOTES FOR THE USER

1. **Guesty listing IDs**: Find these in Guesty → Listings → click a listing → the ID is in the URL
2. **SerpAPI**: Free tier is 100 searches/month. Paid plans start at $50/mo. Highly recommended.
3. **ScraperAPI**: Free tier includes 1,000 credits. Use if Airbnb/VRBO blocks direct requests.
4. **Airbnb scraping**: Airbnb actively blocks scrapers. Without ScraperAPI, you may see reduced results. The system handles this gracefully with fallbacks.
5. **Rate limits**: The system includes built-in delays (2–5 sec between requests) to avoid bans.
6. **Adding more units**: Edit `COMPOSITE_UNITS` in `config.py` to add more Guesty listings.
7. **Adjusting markup**: Change `DESIRED_MARKUP_PCT` in `config.py` (default: 0.20 = 20%).
8. **Adjusting blackout threshold**: Change `min_available_per_source` in each `CompositeUnit` in `config.py`.
